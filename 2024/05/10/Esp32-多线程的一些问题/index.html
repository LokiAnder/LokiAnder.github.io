Esp32 多线程的一些问题

前言

这两天在做esp32的嵌入式方向的设备，期间发现一些相关的一些需要注意的点，在这里记录一下。

FreeRtos中队列阻塞问题

由于我的开发平台是arduino，可以很轻松的使用freertos，同时程序里又用到了队列，所以我便当仁不让的用了freertos中的队列数据结构。本来呢，这个队列数据结构是用于任务之间的数据传输用的，特别是在生产者-消费者问题中非常有用。而我呢，单纯看在他作为队列的数据结构，省的自己写一个队列了，没想着这个队列在freertos的环境下会有阻塞的问题，其最直接的结果就是，我程序一开始跑起来了，但是没跑几步就不见了，就不见了！！！！！

简单来说就是没声了，我一开始以为是跑到哪个线程中的死循环里去了吗？？或者说有哪个线程耗时太长了，把其他线程都给挤死啦？？？

但最后才发现原来是这个队列的锅，这里为了以后的方便，顺便积累一下freertos环境中队列的使用：

基本概念

在 FreeRTOS 中，队列允许你在多个任务之间发送消息（数据项），这些消息可以是任何数据类型，如整数、指针、数组或者更复杂的结构体。每个队列具有固定的长度和项大小，这些在创建队列时被确定。

创建和使用队列

要在 FreeRTOS 中使用队列，你需要执行以下步骤：

  • 创建队列：使用 xQueueCreate() 函数创建队列。
  • 发送数据到队列：可以使用 xQueueSend(), xQueueSendToFront(), xQueueSendToBack(), 和 xQueueSendFromISR() 等函数。
  • 从队列接收数据：可以使用 xQueueReceive() 和 xQueueReceiveFromISR() 等函数。

队列函数

下面是一些常用的队列相关函数：

  • xQueueCreate():

    创建一个新队列。uxQueueLength 是队列可以持有的元素数量，uxItemSize 是每个元素的大小（字节）。

  • xQueueSend():

    向队列发送一个元素。如果队列已满，调用任务将等待指定的时间。

    顺便一提，出问题的就是这个xQueueSend，由于我以前一直用的是循环队列，不用考虑溢出的问题，而且一般编程的话一旦出现溢出会报错，但是在freertos中，它居然是阻塞！！！是阻塞！！！这难不成是期望其他核上的线程来救他嘛？？？😂😂😂😂

  • XQueueReceive():

    从队列接收一个元素。如果队列为空，调用任务将等待指定的时间。

ESP32的核心数

由于这次我设计的程序要求读取传感器数据，做一些运算同时还需要把获得的数据上传到后端，由于几个传感器都需要频繁采集，我就需要让系统随时把资源给到传感器采集的线程。而同时，要将数据上传这个步骤非常的耗时，而且不适合暂停，所以我一开始在做架构的时候还准备再加入一个单片机，用来专门做数据采集的工作，但是如果要再加入一个单片机的话就要考虑同步异步的问题，众所周知，本来在编程中出现线程、时序、锁等等就已经是个大工程，而如果这些问题再出现在嵌入式领域，额。。。。那就是灾难，耗时耗力。虽然那么麻烦，我当时是想“这也没办法嘛”。由于我之前用的一般是arm架构下的单片机，比如说stm32f407、stm32f103之类的，这些都是单核处理器，所以一开时就没多想。现在我知道了！！！！！

敲重点！！！

ESP32-S3 是由 Espressif Systems 开发的一款 Wi-Fi 和蓝牙双模系统级芯片（SoC），是 ESP32 系列的一部分，针对更高的性能和更丰富的功能进行了优化。ESP32-S3 在 ESP32 的基础上进行了多方面的改进，包括更强的处理能力、更多的 IO 口、以及增强的安全特性等。

核心架构

ESP32-S3 使用了两个 Xtensa® 32-bit LX7 微处理器核心。这两个核心可以独立处理任务，也可以协同工作，提供高效的多任务处理能力。Xtensa LX7 是比较先进的处理器架构，相较于 ESP32 使用的 LX6 核心，LX7 提供了更好的性能和更低的功耗。此外，ESP32-S3 的 CPU 频率可达到 240 MHz。

如何分配两个核心的工作！！

这里就需要介绍xTaskCreatePinnedToCore函数了，xTaskCreatePinnedToCore 是 ESP32 环境中 FreeRTOS 的一个函数，用于创建一个任务并将其固定（绑定）到指定的 CPU 核心上。ESP32 是一个双核处理器，有两个可用的核心（通常编号为核心 0 和核心 1）。使用 xTaskCreatePinnedToCore 可以把任务明确分配给特定的核心执行，这有助于任务管理和性能优化。

xTaskCreatePinnedToCore的参数

函数原型如下：

各个参数的具体含义如下：

  • pvTaskCode：任务函数的指针。这是任务开始执行时调用的函数。该函数必须具备 void func(void *param) 的形式。
  • pcName：任务的文本名字。这个名字不会用于核心的调度功能，但可用于调试目的。
  • usStackDepth：任务堆栈的大小，以字为单位。字的大小依据编译器和CPU架构可能有所不同（在 ESP32 中通常是 4 字节）。
  • pvParameters：传递给任务的参数，可以是指向任何类型的数据的指针。这个指针会被传递给任务函数。
  • uxPriority：任务的优先级。在 FreeRTOS 中，较高的数值表示较高的优先级。所有任务都应该具有一个优先级，系统会优先调度优先级更高的任务。
  • pxCreatedTask：任务句柄的指针。如果不需要使用任务句柄，可以设置为 NULL。任务句柄可以用于以后对任务的引用，例如删除任务或更改其优先级等。
  • xCoreID：指定任务应该运行的核心 ID。ESP32 的核心通常编号为 0 和 1。如果你将此参数设置为 tskNO_AFFINITY（通常定义为 -1），任务将不会被固定到任何特定的核心，而是由操作系统调度到任意一个可用核心。
